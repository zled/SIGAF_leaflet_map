When debugging web server code with Visual studio both IIS Express and browser run on the same computer and IIS is connected to LOCALHOST.
Localhost entry in c:\\windows\\system32\\etc\\hosts, links with IPv6 ::1 and IPv4 127.0.0.1 . Both Request.ServerVariables[\"REMOTE_ADDR\"] and Request.ServerVariables[\"LOCAL_ADDR\"] contains the IPv6 address of LOCALHOST, ::1. So it is not possible to recognise if you are working from my workorganization (ARTEA) intranet or from home. I buid an c# server code to detect the IPv4 address of computer hosting IIS Express web server when coding and debugging with Visual Studio. It uses the System.Net.Dns GetHostName() and GetHostEntry() methods. The public static System.Net.IPHostEntry GetHostEntry (string hostNameOrAddress) produces a list of all the ip address of computer coniaining the same addresses that we found in the output of \"ipconfig /all\" command from command prompt. In my work intranet user's network are 192.168.60.0/24, 192.168.72.0/24, 192.168.85/24.  VPN connecter pc's have in list both the home IP address  (in most cases 192.168.1.0/24) and the intranet address (192.168.72.0/24). For this reason I scan all the list and when finding the 72/24 address I definetly consider to be in the work intranet.
